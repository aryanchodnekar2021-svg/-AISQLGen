# 🤖 AISQLGen

> **Stop writing SQL queries by hand.** Connect directly to your local or remote database, type your question in plain English, and let AI generate SQL queries, edit existing scripts, and auto-generate interactive visualizations. Run it completely locally.

---

## 🎨 Fresh Look
We've updated the theme of **AISQLGen** to a clean, premium **forest green palette** matching `#235347` (primary brand color) for a modern, sleek interface.

---

## 🚀 Key Features

*   **Natural Language to SQL**: Generate complex queries, joins, and aggregations using simple, conversational prompts.
*   **Interactive Editing**: Refine and tweak generated SQL by hand or ask the AI to modify specific parts (e.g., adding `WHERE` clauses, sorting, or grouping).
*   **Auto-Generated Charts**: Instant data visualization. Convert query results into interactive charts (powered by Chart.js) directly inside the tool.
*   **Privacy & Security First**: All database credentials, API keys, and query histories are saved locally in your browser/filesystem. No telemetry or server-side sharing.
*   **Multi-Database Support**: Out-of-the-box support for Postgres, ClickHouse, and SQLite.
*   **Query History**: Trace back your query runs, prompt iterations, and previous versions in a structured history panel.

## ⚡ Quick Start

### Prerequisites
*   **Node.js**: v18.0.0 or later (Tested on Node v24)
*   **Yarn**: Required for dependency installation (Workspaces)
*   **OpenAI API Key**: For query generation and chart rendering
*   **Database Instance**: A running instance of PostgreSQL, SQLite, or ClickHouse

### 1. Install Dependencies
Run the following command at the root directory of the project:
```bash
yarn install
```

### 2. Run the Development Server
You can start the frontend app either from the root or by navigating to the frontend folder:

**From the Root:**
```bash
yarn workspace frontend dev
```

**Or From the Frontend Folder:**
```bash
cd frontend
yarn dev
```

### 3. Open the App
Go to **[http://localhost:3000](http://localhost:3000)** in your browser.

## ⚙️ Configuration & Usage Flow

### Step 1: Set Up your OpenAI API Key
Upon opening the app, a modal will prompt you to enter your OpenAI API key. This key is saved locally in your browser's IndexedDB.

![OpenAI Key Setup](docs/screenshot-api-key.png)

### Step 2: Connect a Data Source
Click on **Add Datasource** in the left sidebar and configure your database type and connection string. You can choose from **Postgres**, **SQLite**, or **ClickHouse**.

![Add Datasource](docs/screenshot-add-datasource.png)

### Step 3: Ask questions & Generate SQL
Type your question in natural language and click **Generate**. Review the SQL query generated by the AI model.

![Query Generation](docs/screenshot-query-generated.png)

### Step 4: Edit & Execute
Need changes? Ask the AI to edit your query (e.g., *"Filter where status is active"*), or directly type and correct the SQL syntax manually inside the Monaco editor. Click **Run Query** to see database outputs.

![Editing Queries](docs/screenshot-query-edit-end.png)

### Step 5: Visualize your Data
Click **Generate Chart** to automatically create visualizations for your query result tables.

![Chart Visualization](docs/screenshot-query-chart.png)

## 🏗️ Architecture

All data flow is handled locally on your machine. The Next.js API routes run local Node.js database connectors, querying your database directly and sending schema layouts to OpenAI to construct correct SQL scripts.

```
                  ┌──────────────────────┐
                  │   Browser Frontend   │
                  │   (Next.js / React)  │
                  └──────────┬───────────┘
                             │ (API request)
                             ▼
                  ┌──────────────────────┐
                  │ Local Next.js Server │
                  └────┬────────────┬────┘
                       │            │
         (Query Schema)│            │(Local Connection)
                       ▼            ▼
             ┌───────────┐        ┌────────────┐
             │OpenAI API │        │Your Local  │
             │   (LLM)   │        │  Database  │
             └───────────┘        └────────────┘
```
